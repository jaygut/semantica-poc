# Scenario Audit Bundle

This directory contains 5 canonical scenario transcripts generated by the v6 Scenario Intelligence engine. Each transcript is a complete, reproducible record of a scenario computation - from request to response to validation.

## Purpose

The audit bundle exists to:
1. **Demonstrate determinism** - every scenario output traces to bridge axiom coefficients and peer-reviewed sources
2. **Validate against empirical anchors** - each transcript includes a validation_anchors block that checks computed values against known ranges from literature or market data
3. **Enable independent review** - investors, auditors, or regulators can inspect the full propagation trace without running the system

## Transcript Structure

Each JSON file contains:

```json
{
  "transcript_id": "unique identifier",
  "generated_at": "ISO 8601 timestamp",
  "scenario_request": { "scenario_type": "...", "site_scope": [...], ... },
  "scenario_response": { "baseline_case": {...}, "scenario_case": {...}, ... },
  "validation_anchors": {
    "expected_range": "human-readable description",
    "actual_value": 123456.78,
    "within_range": true
  },
  "confidence_breakdown": [...],
  "uncertainty": { "p5": ..., "p50": ..., "p95": ... }
}
```

## How to Regenerate

```bash
python scripts/generate_scenario_audit_bundle.py
```

The script requires no external services (no Neo4j, no LLM). It reads case study data from `examples/` and computes scenario outputs using the deterministic engines in `maris/scenario/`.

All 5 transcripts must show `within_range: true` in their validation_anchors block. If any fail, the script exits with code 1.
